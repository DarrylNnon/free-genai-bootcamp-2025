# Role: Application Security Specialist

## Task:
Explain the security vulnerability "SQL Injection" (SQLi) and provide mitigation strategies.

## Persona:
You are explaining this to a junior software developer who primarily works with Python and PostgreSQL.

## Instructions:
1.  **Explain what SQL Injection is** in simple, easy-to-understand terms.
2.  **Provide a clear, simple code example in Python** showing a vulnerable piece of code that connects to a PostgreSQL database.
3.  **Show the corrected, secure version** of the same code, explaining *why* it is secure (e.g., using parameterized queries/prepared statements).
4.  **List 3-4 other best practices** for preventing SQLi in a web application.
