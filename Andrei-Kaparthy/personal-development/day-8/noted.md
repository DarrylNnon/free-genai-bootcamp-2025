## The "Assembly Line" Guide to AI-Driven Automation

Think of today's project not as a single "shot," but as an assembly line. Each station performs a specific task, handing its output to the next. By chaining these simple steps, we create a powerful and automated workflow that embodies the principles of DevSecOps.

---

### Station 1: The User Request (Input)

The entire process starts with a simple, human-friendly request. This is the "order" for our assembly line.

**Real-World Scenario:** A developer needs a new, secure S3 bucket for storing sensitive logs but isn't a Terraform expert. Instead of looking up documentation, they just ask the assistant.

**How it works:**
The user runs the Python script from the command line, passing their request as an argument.

**Implementation (`iac_assistant.py`):**

```

### Station 2: Prompt Engineering (GPT)

This is the core "intelligence" of our assistant. We take the user's simple request and translate it into a detailed, structured prompt for the LLM. This is where we inject our security requirements.

**Real-World Scenario:** We need to ensure the S3 bucket generated by the AI is not just functional but also secure by default (e.g., private, encrypted, versioned).

**How it works:**
The script reads a template prompt and inserts the user's request. This "few-shot" prompt guides the LLM to produce high-quality, secure HCL (HashiCorp Configuration Language).

**Implementation (`iac_assistant.py` calling `prompts/secure_iac_generator.md`):**
```python
# A simplified view of the prompt template
"""
<PERSONA>
You are a top-tier DevSecOps engineer who specializes in writing secure Terraform code.
</PERSONA>

<TASK>
Generate a secure Terraform configuration for the user's request.
The code must be secure by default. For example, S3 buckets must have versioning, encryption, and no public access.
</TASK>

<CONTEXT>
User's request: "{user_request}"
</CONTEXT>

<OUTPUT_FORMAT>
Provide the response as a single Terraform code block.
</OUTPUT_FORMAT>
```
```

### Station 3: Security Scanning (tfsec)

Trust, but verify. We can't blindly trust the code generated by the LLM. This station acts as our automated quality control, scanning the generated Terraform code for common security misconfigurations.

**Real-World Scenario:** The LLM generates code for an S3 bucket but forgets to enable server-side encryption. `tfsec` will catch this before it ever gets deployed.

**How it works:**
The Python script saves the LLM's output to a `.tf` file and then runs the `tfsec` command-line tool as a subprocess. The results are saved to a JSON file for the next station.

**Implementation (`iac_assistant.py`):**
```python
def run_tfsec():
    """Runs tfsec on the current directory and handles its output."""
    command = ["tfsec", ".", "--format", "json", "--out", "tfsec-results.json"]
    try:
        # tfsec exits with non-zero status if issues are found. This is expected.
        subprocess.run(command, check=False, capture_output=True, text=True)
    except FileNotFoundError:
        print("Error: 'tfsec' command not found.")
        sys.exit(1)
```

### Station 4: AI-Powered Reporting (Report)

The final station closes the loop. Raw JSON output from `tfsec` can be cryptic. We use the power of the LLM again to translate this technical data into a clear, actionable report for the developer.

**Real-World Scenario:** `tfsec` flags an issue. Instead of just showing a JSON error, the assistant explains *why* it's a risk and suggests the exact code change needed to fix it.

**How it works:**
The script reads the `tfsec` JSON output and constructs a new prompt. This prompt asks the LLM to act as a DevSecOps expert, interpret the scan results, and create a final summary for the user.

**Implementation (`iac_assistant.py`):**
```python
def generate_final_report_prompt(user_request, terraform_code, tfsec_results_json):

</PERSONA>

<TASK>
Your task is to create a final, human-readable report for the user in Markdown format.
...
</TASK>

<CONTEXT>
- **User's Original Request:** "{user_request}"
- **Generated Terraform Code:** ...
- **Security Scan Analysis Task:** ...
</CONTEXT>
"""
    return prompt.strip()
```
