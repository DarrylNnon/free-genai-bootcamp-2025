# IaC GPT Assistant

Welcome to the IaC GPT Assistant, a project from Day 11 of the "LEGENDARY 60-Day GenAI DevSecOps Engineer" challenge. This tool embodies AI-Native DevSecOps by automating the generation and security validation of Infrastructure as Code (IaC).

This tool uses a multi-step, AI-driven "assembly line" to turn a natural language request into secure, validated Terraform code.

## 🚀 Features

- **Natural Language to IaC**: Describe your infrastructure needs in plain English (e.g., "a secure S3 bucket for private logs").
- **Security-First Generation**: The assistant is prompted to generate Terraform code that follows security best practices by default.
- **Automated Security Scanning**: It automatically runs `tfsec` to scan the generated code for misconfigurations.
- **AI-Powered Prompt Chaining & Auto-Remediation**: If `tfsec` finds issues, the assistant uses a second AI prompt to *attempt to fix the code automatically*.
- **Comprehensive Reporting**: A final, human-readable report is generated by an AI, explaining the generated code, the security scan results, and any fixes applied.
- **Polished Web UI**: A simple and clean web interface built with Flask and a touch of CSS for a smooth user experience.

## ⚙️ The Assembly Line Workflow

The assistant follows a sophisticated, chained workflow:

1.  **User Input**: You provide a request through the web UI.
2.  **Initial Generation (GPT-4)**: The first AI prompt generates a Terraform configuration based on your request, with a strong emphasis on security.
3.  **First Security Scan (`tfsec`)**: The generated code is saved and scanned by `tfsec`.
4.  **Analyze & Fix (GPT-4)**:
    - **If issues are found**: A second "fixer" prompt is sent to the AI, containing the problematic code and the `tfsec` report. The AI attempts to provide a corrected version.
    - **If no issues are found**: This step is skipped.
5.  **Final Security Scan (`tfsec`)**: If the code was fixed, it's scanned again to verify the remediation.
6.  **Final Report (GPT-4)**: A final AI prompt synthesizes all information—the original request, the final code, and the final scan results—into a clear, actionable Markdown report.
7.  **Display Results**: The final code and report are displayed in the web UI.

## 🛠️ Setup and Installation

Follow these steps to get the IaC GPT Assistant running locally.

### Prerequisites

- Python 3.8+
- Terraform installed and in your `PATH`.
- `tfsec` installed and in your `PATH`.

### 1. Clone the Repository

If you haven't already, clone the project to your local machine.

### 2. Set Up a Virtual Environment

```bash
python -m venv .venv
source .venv/bin/activate
# On Windows, use: .venv\Scripts\activate
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure Environment Variables

Copy the example environment file:
```bash
cp .env.example .env
```
Now, edit the `.env` file and add your OpenAI API key:
```
OPENAI_API_KEY="sk-..."
```

### 5. Run the Application

```bash
flask run
```

Open your browser and navigate to `http://127.0.0.1:5000`. You're ready to go!
